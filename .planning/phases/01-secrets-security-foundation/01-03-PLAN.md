  <action>
    Create setup_users_table.py at the project root. The script must:

    1. Call load_dotenv() at the top (imports: os, bcrypt, from dotenv import load_dotenv, from db_marks import get_connection)

    2. Read ADMIN_USER and ADMIN_PASS from environment. Add a guard:
       ```python
       admin_user = os.getenv("ADMIN_USER")
       admin_pass = os.getenv("ADMIN_PASS")
       if not admin_user or not admin_pass:
           raise EnvironmentError("ADMIN_USER and ADMIN_PASS must be set in .env before running this script")
       ```
       This prevents accidentally seeding with an empty password if .env is incomplete.

    3. Hash the admin password using bcrypt:
       ```python
       hashed = bcrypt.hashpw(admin_pass.encode("utf-8"), bcrypt.gensalt())
       password_hash = hashed.decode("utf-8")
       ```

    4. Connect to MySQL using get_connection() from db_marks. Execute CREATE TABLE IF NOT EXISTS:
       ```sql
       CREATE TABLE IF NOT EXISTS users (
           id            INT AUTO_INCREMENT PRIMARY KEY,
           username      VARCHAR(100) NOT NULL UNIQUE,
           password_hash VARCHAR(60)  NOT NULL,
           role          ENUM('admin', 'faculty') NOT NULL,
           is_active     BOOLEAN DEFAULT TRUE,
           created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP
       )
       ```

    5. Insert the admin row using INSERT IGNORE (idempotent — safe to re-run):
       ```python
       cursor.execute(
           "INSERT IGNORE INTO users (username, password_hash, role) VALUES (%s, %s, 'admin')",
           (admin_user, password_hash)
       )
       connection.commit()
       ```

    6. Print a confirmation message: "users table ready. Admin account '{admin_user}' seeded (or already exists)."

    7. Wrap the DB operations in try/finally to ensure connection.close() is always called.

    Anti-patterns to avoid:
    - Do NOT use os.getenv("ADMIN_PASS", "some_default") — no fallback password in code.
    - Do NOT pass a str to bcrypt.hashpw() — always .encode("utf-8") first (bcrypt 5.0.0 requires bytes input; passing str raises TypeError).
    - Do NOT store the plaintext admin_pass anywhere — only the hashed value touches the database.
